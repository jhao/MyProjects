# UI/UX 设计标准

本项目采用清爽、紧凑、偏工作台式的信息架构。页面优先服务高频扫描、检索、更新和归档，不采用营销型大图或低密度卡片布局。

## 布局
- 顶部导航高度 56-64px，左侧为 Logo 和系统名称，右侧为头像、昵称、退出。
- 普通用户首页无侧边栏、无多余顶层菜单，主区域直接展示项目列表。
- 管理后台可使用顶部二级标签或左侧窄导航。
- 详情面板优先使用右侧抽屉，保留列表上下文。
- 桌面端内容宽度尽量铺满，保留 20-24px 页面边距。
- 移动端表格切换为紧凑卡片或横向滚动。

## 字体与密度
- 字体栈：`-apple-system, BlinkMacSystemFont, "Segoe UI", "PingFang SC", "Microsoft YaHei", Arial, sans-serif`。
- 正文字号 13-14px，辅助信息 12px，页面标题 18-22px。
- 表格行高 44-52px，紧凑但不拥挤。

## 色彩
- 主色：#2563eb。
- 背景：#f8fafc。
- 面板背景：#ffffff。
- 边框：#e2e8f0。
- 正文：#111827。
- 次文字：#64748b。
- 成功：#16a34a；警告：#d97706；错误：#dc2626；信息：#2563eb。
- 分类和状态可使用不同色相，但同屏避免单一蓝色占满。

## 组件
- 表格用于主列表，支持筛选、排序、分页。
- 状态用浅底色标签，多状态并列展示。
- 进度用细进度条 + 数字。
- 危险操作必须二次确认。
- 文件目录使用树形结构。
- 日志使用周/月切换 + 日期网格 + 富文本编辑区。

## 交互
- 关键按钮点击后给出 toast 反馈。
- 抽屉支持关闭、标签切换。
- 表单字段必须清楚标注必填。
- 搜索结果需要展示命中来源。
- 原型阶段允许使用前端假数据模拟交互。
